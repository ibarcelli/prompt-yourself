# Prompt Yourself — Pipeline

## Tipo de Proyecto
Producto digital (PWA) — app de priming y visualización personalizada con motor de IA.

## Fases

### Fase 0: Fundación ✅
- [x] Semilla completada
- [x] Consejo completado
- [x] Brain generado
- [x] Backlog creado

### Fase 1: POC — Prompt Engine 🔲
**Objetivo**: Generar un prompt diario personalizado para Ives basado en su goal alignment y perfil.
**Definition of Done**: Ives recibe un prompt personalizado que cumple con la estructura de 4 partes y se siente como su propio inconsciente hablándole.
- [ ] Diseñar estructura interna del prompt (anclaje, declaración, visualización, cierre)
- [ ] Crear system prompt para el motor de IA que genere prompts personalizados
- [ ] Definir mapa de dimensiones humanas con gaps
- [ ] Probar con perfil de Ives — generar 7 prompts (1 semana)
- [ ] Ives valida: ¿esto se siente como mi inconsciente o como un coach?

### Fase 2: POC — Onboarding 🔲
**Objetivo**: Crear el flujo de perfilamiento que captura la esfera del usuario.
**Definition of Done**: Un usuario nuevo puede completar el onboarding en 5-7 min y la app genera su primer prompt relevante.
- [ ] Diseñar preguntas de nivel de avance metafísico
- [ ] Diseñar preguntas de contexto de vida
- [ ] Diseñar flujo de estado deseado → lenguaje declarativo
- [ ] Implementar selector de marco (científico vs metafísico)
- [ ] Probar con Gisella y Fiorella (o perfiles simulados)

### Fase 3: MVP — Interfaz 🔲
**Objetivo**: PWA funcional donde el usuario abre, recibe su prompt, lo practica y cierra.
**Definition of Done**: App desplegada, accesible desde navegador móvil, instalable como PWA.
- [ ] Pantalla de onboarding
- [ ] Pantalla diaria (el prompt del día)
- [ ] Pantalla de perfil básico
- [ ] Diseño minimalista — sin menús innecesarios
- [ ] Despliegue en hosting gratuito

### Fase 4: Iteración con Usuarios 🔲
**Objetivo**: 10-20 usuarios reales usando la app diariamente y dando feedback.
**Definition of Done**: Feedback de al menos 10 usuarios, métricas de retención diaria, iteraciones implementadas.
- [ ] Invitar beta testers (red de Ives, Gisella, socios)
- [ ] Implementar tracking básico de uso
- [ ] Recoger feedback cualitativo
- [ ] Iterar prompts y onboarding según feedback

### Fase 5: Monetización 🔲
**Objetivo**: Modelo freemium funcionando con pagos.
**Definition of Done**: Al menos 5 usuarios pagando.
- [ ] Definir qué es gratis vs premium
- [ ] Integrar pagos (Stripe o equivalente LATAM)
- [ ] Definir precio final basado en datos del POC

### Fase 6: Audio + Sofisticación 🔲
**Objetivo**: Agregar audio con frecuencias binaurales y voz sintética.
**Definition of Done**: Usuarios premium reciben audio diario además de texto.
- [ ] Evaluar ElevenLabs o alternativa para voz
- [ ] Diseñar capas de frecuencias (10 Hz alfa, 6 Hz theta)
- [ ] Integrar generación de audio al flujo diario
